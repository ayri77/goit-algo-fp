# Завдання 5: Обхід бінарного дерева

## Опис

Візуалізація обходів дерева в глибину (DFS) та в ширину (BFS) з кольоровим кодуванням. Використовується стек та черга (без рекурсії).

## Реалізація алгоритмів

### DFS (Depth-First Search) - Обхід в глибину

**Реалізація:** Використовує стек (LIFO - Last In First Out)

**Алгоритм:**
1. Додаємо корінь у стек
2. Поки стек не порожній:
   - Витягуємо вузол зі стеку
   - Додаємо вузол до результату
   - Додаємо правого, потім лівого нащадка у стек
3. Повертаємо список вузлів у порядку обходу

**Порядок обходу:** Корінь → Лівий піддерево → Правий піддерево (Pre-order)

### BFS (Breadth-First Search) - Обхід в ширину

**Реалізація:** Використовує чергу (FIFO - First In First Out)

**Алгоритм:**
1. Додаємо корінь у чергу
2. Поки черга не порожня:
   - Витягуємо вузол з черги
   - Додаємо вузол до результату
   - Додаємо лівого, потім правого нащадка у чергу
3. Повертаємо список вузлів у порядку обходу

**Порядок обходу:** Рівень за рівнем, зліва направо

## Порівняння DFS та BFS

### Приклад дерева

```
        1
      /   \
     2     3
    / \   / \
   4   5 6   7
```

### Порядок обходу DFS (зі стеком)

```
Крок 1: [1] → обробляємо 1
Крок 2: [3, 2] → обробляємо 2
Крок 3: [3, 5, 4] → обробляємо 4
Крок 4: [3, 5] → обробляємо 5
Крок 5: [3] → обробляємо 3
Крок 6: [7, 6] → обробляємо 6
Крок 7: [7] → обробляємо 7

Результат: [1, 2, 4, 5, 3, 6, 7]
```

### Порядок обходу BFS (з чергою)

```
Крок 1: [1] → обробляємо 1
Крок 2: [2, 3] → обробляємо 2
Крок 3: [3, 4, 5] → обробляємо 3
Крок 4: [4, 5, 6, 7] → обробляємо 4
Крок 5: [5, 6, 7] → обробляємо 5
Крок 6: [6, 7] → обробляємо 6
Крок 7: [7] → обробляємо 7

Результат: [1, 2, 3, 4, 5, 6, 7]
```

## Таблиця порівняння

| Характеристика | DFS | BFS |
|----------------|-----|-----|
| Структура даних | Стек | Черга |
| Порядок обходу | Глибина спочатку | Ширина спочатку |
| Складність часу | O(V + E) | O(V + E) |
| Складність пам'яті | O(h), де h - висота | O(w), де w - ширина |
| Застосування | Пошук у глибину, топологічне сортування | Пошук найкоротшого шляху, рівні дерева |

## Візуалізація

Візуалізація показує:
- **Структуру дерева**: Зв'язки між вузлами
- **Кольорове кодування**: Від темних до світлих відтінків залежно від порядку обходу
- **Порядок обходу**: Легко побачити різницю між DFS та BFS

### DFS візуалізація
- Вузли, оброблені раніше, мають темніші кольори
- Видно "глибокий" характер обходу

### BFS візуалізація
- Вузли одного рівня мають схожі кольори
- Видно "широкий" характер обходу

## Висновки

1. **Коректність**: Обидва алгоритми працюють правильно без рекурсії
2. **Різниця в обході**: DFS та BFS дають різні порядки обходу вузлів
3. **Кольорове кодування**: Ефективно показує порядок обходу
4. **Застосування**: 
   - DFS: для пошуку в глибину, перевірки циклів
   - BFS: для пошуку найкоротшого шляху, обходу рівнями

**Спостереження:**
- DFS використовує менше пам'яті для "високих" дерев
- BFS використовує менше пам'яті для "широких" дерев
- Обидва алгоритми мають однакову часову складність
